# Measurement — Protein Map / 蛋白质地图

How this site is instrumented and what to look at. Generated by `build.py`.

## North-star metric
**M0 — Monthly organic clicks**: total clicks from Google Search to any page, measured in
Google Search Console (Performance report, search type = Web). This is the single number
that says whether the SEO strategy is working.

## Supporting metrics
| ID | Metric | Where to read it |
|----|--------|------------------|
| M1 | Indexed-page count | Search Console → Pages (Indexed) vs the ~251 URLs built |
| M2 | Cluster impressions + average position | Search Console → Performance → Queries (filter: protein / plant protein / DIAAS / <amino> foods) |
| M3 | Tool-open rate | GA4: `tool_cta_click` events ÷ entry-page sessions; plus `search_used` / `filter_used` as in-tool engagement |
| M4 | Food-page organic clicks | Search Console → Performance → Pages (filter `/foods/`) |
| M5 | zh organic clicks | Search Console → Performance → Pages (filter `/zh/`) |
| M6 | Health-claim leakage | build-time check (see "Compliance checks") — target 0 |

## Analytics setup
GA4 status for this build: **not set — analytics is OFF until `MEASUREMENT_ID` is exported at build time**.

The GA4 tag is injected in `<head>` **only when `MEASUREMENT_ID` is set** at build time, so
local/dev builds send no data. To turn it on in production:

```
MEASUREMENT_ID="G-XXXXXXXXXX" SITE_URL="https://<your-domain>" python3 build.py
```

Events are sent via `static/analytics.js`, which defines a small `window.track(name, params)`
helper (a no-op when GA4 is absent). Core interactions in `static/app.js` call it; site-wide
links fire through `[data-event]` attributes handled by delegation.

## Event dictionary
| Event | Fires when | Key params | Metric |
|-------|-----------|------------|--------|
| `search_used` | visitor types in the table search box (debounced) | `q_len` | M3 |
| `filter_used` | category filter changes, or a column header is sorted | `type` (category/sort), `key` | M3 |
| `food_page_open` | a `/foods/*` page loads (incl. direct organic landings) | `slug` | M4 |
| `compare_clicked` | a compare checkbox toggles, or compare is cleared | `action` | M3 |
| `language_switch` | the EN↔中文 switcher is clicked | `to` | M5 |
| `source_link_click` | an outbound data-source / reference link is clicked | `label` | M4/trust |
| `tool_cta_click` | "Open the interactive table" CTA on a door page is clicked | `from` | M3 (funnel) |

`food_page_open` is the only auto-fired (load) event; the rest are interactions.
`tool_cta_click` is the primary funnel signal for M3 (added beyond the original six because
M3 depends on it).

## Compliance checks (run every build, in `validate_pages()`)
Every indexable page must contain: a `canonical` link, `hreflang` alternates, a non-empty
`meta description`, and a `disclaimer`. A noindex page (the 404) is exempt. If any indexable
page is missing one, **the build fails** (non-zero exit) and prints which page and what's
missing — fix before deploying.

## Google Search Console setup (one-time)
1. Go to search.google.com/search-console and add a **URL-prefix** property for the live site
   URL (currently `https://wenw121.github.io/foodmap-site/`).
2. Choose **HTML tag** verification. Copy the `content="..."` token it gives you.
3. Add it to `templates/base.html` `<head>` as
   `<meta name="google-site-verification" content="...">`, then rebuild and deploy.
4. Back in Search Console, click **Verify**.
5. Submit the sitemap: `https://wenw121.github.io/foodmap-site/sitemap.xml` under **Sitemaps**.
6. Use **URL Inspection** → Request indexing for the homepage and a few key hub pages.

GA4 and Search Console can be linked (GA4 Admin → Product links → Search Console) so query
data shows up alongside event data.
